# astrbot_plugin_group_relations

给 AstrBot 使用的群关系上下文插件。它不是靠用户日常输入指令来工作，而是在每次 LLM 请求前，自动根据当前群、当前发言人和本轮消息召回少量人物关系，并作为临时上下文注入给模型。

## 核心逻辑

1. 写入关系时保存结构化三元组：`主体 --关系--> 客体`。
2. 每个群/私聊会被记录为独立“群空间”，同一群的关系和用户画像默认汇总到同一个空间。
3. 每个群空间下按用户 ID 动态维护用户画像，包括昵称/群名片、别名、触达次数和稳定画像事实。
4. 每条关系使用 AstrBot 已配置的 Embedding Provider 向量化。
5. 每次 LLM 请求前，用 `群名/私聊标识 + 群空间 ID + 发言人昵称/群名片 + 发言人 ID + 本轮消息` 组成查询文本。
6. 默认按群隔离记忆，群聊按群统一管理，私聊仍按会话隔离。
7. 召回到的少量关系、当前群空间状态、当前会话身份说明和当前发言人的画像会通过 `req.extra_user_content_parts` 临时注入，不写入会话历史。
8. LLM 需要更多信息时，可以主动调用工具搜索关系或用户画像。
9. 用户明确纠正时，LLM 可以在配置允许的情况下写入、修改或删除关系，也可以写入稳定用户画像。
10. 可选开启每隔几轮对话自动总结，用指定 LLM Provider 抽取关系和人物画像事实。

## 调试指令

```text
/关系 状态
/关系 群
/关系 用户 [用户ID或昵称]
/关系 画像 [用户ID或昵称]
/关系 删除画像 <用户ID或昵称> [画像关键词]
/关系 调试 <查询词>
/关系 最近
/关系 向量
```

这些指令只用于开关检查和 debug，不是主要交互入口。默认只有配置里的管理员或平台可识别的管理员可以使用，避免把群关系记忆暴露给普通成员。

## WebUI 管理页

插件提供 AstrBot 插件 Pages 管理页：`group-memory`。页面会按群空间展示关系和用户画像，支持修改群空间名称、增删改关系、增删改用户画像和画像事实。

后端接口通过 `context.register_web_api` 注册，页面通过 `window.AstrBotPluginPage` 调用，不需要单独启动服务。

## LLM 工具

- `group_relation_search`：主动搜索当前群人物关系。
- `group_user_profile_search`：主动搜索当前群用户画像。
- `group_relation_remember`：写入当前群人物关系，受 `enable_tool_write` 控制。
- `group_user_profile_remember`：写入当前群用户画像事实，受 `enable_tool_write` 控制。
- `group_relation_update`：修改当前群人物关系，受 `enable_tool_update` 控制。
- `group_relation_delete`：删除当前群人物关系，受 `enable_tool_update` 控制。

## 重要配置

- `enable_context_injection`：是否每轮自动注入关系上下文，默认开启。
- `enable_dialogue_summary`：是否每隔几轮对话自动总结并写入关系，默认关闭。
- `自动总结_触发轮数`：自动总结触发轮数，默认 6。
- `自动总结_模型Provider`：自动总结使用的 LLM Provider，留空使用当前会话模型。
- `自动总结_人格选择`：自动总结时参考的 AstrBot 已配置人格，用于辅助理解说话风格和称呼，不直接作为记忆事实写入。
- `自动总结_最大对话长度`：每次总结传给模型的最近对话最大长度，默认 4000 字。
- `memory_scope`：记忆隔离范围，默认 `group`，可选 `session` / `group` / `global`。
- `记忆隔离_允许跨群召回`：默认按群分离注入；模型工具查询时优先返回当前群结果，再补充同一用户其他群空间里的记忆。
- `私聊_同步到所属群空间`：私聊中的用户画像会同步到该用户已出现过的群空间，不会凭空创建新群。
- `enable_session_identity_injection`：是否明确注入“当前在哪个群聊/私聊”的会话身份说明，默认开启。
- `bot_relation_aliases`：机器人在关系记忆里的可能称呼，用于检索机器人自己和群友的关系。
- `relation_admin_user_ids`：允许使用 `/关系` 调试指令的用户 ID，多个 ID 可用英文逗号或换行分隔。
- `allow_public_debug_commands`：是否允许所有人使用 `/关系` 调试指令，默认关闭，只建议测试群临时开启。
- `enable_person_profile`：是否生成当前发言人的简易画像，默认开启。
- `记忆管理_每轮注入关系数量`：每轮最多注入多少条相关关系，默认 5。
- `记忆管理_每人关系召回上限`：当前发言人每轮最多召回多少条直接关系，默认 8。
- `记忆管理_当前发言人画像条数`：当前发言人画像最多注入多少条，默认 4。
- `记忆管理_相关人物画像注入人数`：每轮最多注入多少个相关人物画像，默认 4。
- `记忆管理_画像查询返回人数`：画像查询最多返回多少个用户，默认 5。
- `记忆管理_每人记忆上限`：每个用户最多保留多少条画像事实/长期记忆，默认 20。
- `记忆管理_每轮记忆更新操作上限`：每轮总结最多执行多少条记忆更新，默认 6。
- `记忆管理_写入可信度阈值`：总结/抽取结果低于该可信度时不写入，默认 0.65。
- `群身份_初始化扫描`：只在刚创建群空间或当前发言人没有身份记录时归纳群主/管理/群友，默认开启。
- `embedding_provider_id`：AstrBot Embedding Provider。留空时自动使用第一个可用 Provider。
- `enable_tool_read`：是否允许 LLM 主动搜索关系，默认开启。
- `enable_tool_write`：是否允许 LLM 主动写入关系，默认关闭。
- `enable_tool_update`：是否允许 LLM 主动修改/删除关系，默认关闭。
- `enable_auto_extract`：是否自动从群聊消息抽取关系，默认关闭。

建议先开启自动注入和工具搜索；写入、修改、删除能力等你确认模型行为稳定后再打开。没有 AstrBot Embedding Provider 时，插件会保留轻量本地检索能力，避免关系被写入后完全搜不到；如果要获得更好的召回效果，仍建议配置正式的 Embedding Provider。
