# astrbot_plugin_group_relations

云云给 AstrBot 用的群关系记忆插件。它不是靠群友天天敲命令工作的，而是在每次 LLM 请求前，按当前群、当前发言人和本轮消息召回一点点有用的人物关系，再临时塞进上下文里。这样模型回答群友身份、昵称、关系、喜好时不至于两眼一黑

> ⚠️ **本插件由 AI 生成，使用前请自行审查代码与运行效果，重要场景请先小范围测试。**

## 云云现在会记什么

1. 关系记忆：保存成结构化三元组，格式类似 `主体 --关系--> 客体`，并尽量绑定 `subject_user_id` / `object_user_id`。
2. 群空间：群聊固定按群号隔离，私聊按会话独立，不再提供可选隔离模式。
3. 私聊同步：私聊里总结出的画像会同步到该用户已经出现过的所有群空间，不会凭空创建新群。
4. 群成员目录：新群第一次触达时会尝试拉成员列表，写入带 user_id 的画像/关系前会校验是不是本群成员。
5. 成员身份修正：WebUI 可以把成员身份改成 `owner` / `admin` / `member`，手动修正过的身份不会被后续平台刷新随便盖回去。
6. 基础画像：昵称、首选称呼、爱好、讨厌项、稳定特征、长期备注会单独存，不再和普通事实糊成一坨。
7. 普通画像事实：仍然保留原来的事实列表逻辑，用来存不适合放进基础画像但长期有用的信息。
8. 跨群查询：默认注入仍按当前群隔离；模型主动用工具查人时，可以补充同一 user_id 在其他群空间里的记忆。

## 持久化

插件数据放在 AstrBot 数据目录下：

```text
<astrbot_data>/plugin_data/astrbot_plugin_group_relations/relations.json
```

也就是代码里通过 `get_astrbot_data_path()` 取 AstrBot 数据目录，再落到 `plugin_data`。云云没有把运行数据写进插件源码目录，放心，仓库不会被记忆文件污染。保存时会先写临时文件，再用 `os.replace` 原子替换，尽量避免写一半炸了把 JSON 搞坏。

## 调试指令

```text
/关系 状态
/关系 群
/关系 刷新目录
/关系 更新成员列表
/关系 用户 [用户ID或昵称]
/关系 画像 [用户ID或昵称]
/关系 删除画像 <用户ID或昵称> [画像关键词]
/关系 调试 <查询词>
/关系 最近
```

这些指令是给管理员排查用的，不是日常聊天入口。默认只有配置里的管理员或平台能识别的管理员能用，别把群关系记忆直接摊给普通群友看，怪吓人的。

## WebUI 管理页

插件提供 AstrBot Pages 管理页：`group-memory`。

页面可以做这些事：

- 查看和切换群空间
- 修改群空间名称和群主信息
- 查看群成员目录
- 修正成员身份：`owner` / `admin` / `member`
- 新增、修改、删除人物关系
- 编辑用户显示名、首选称呼和别名
- 维护基础画像：喜欢、讨厌、稳定特征、长期备注
- 维护普通画像事实

后端接口通过 `context.register_web_api` 注册，页面通过 `window.AstrBotPluginPage` 调用，不需要单独起服务。

## LLM 工具

- `group_user_context_lookup`：按 `user_id` 查询当前群身份、画像、关系，并可补充跨群记忆。
- `group_relation_save`：写入当前群人物关系，受 `enable_tool_write` 控制。
- `group_user_profile_save`：写入普通用户画像事实，受 `enable_tool_write` 控制。
- `group_user_basic_profile_update`：维护昵称、爱好、讨厌项、稳定特征等基础画像；新增受 `enable_tool_write` 控制，删除/替换受 `enable_tool_update` 控制。
- `group_relation_update`：修改当前群人物关系，受 `enable_tool_update` 控制。
- `group_relation_delete`：删除当前群人物关系，受 `enable_tool_update` 控制。

## 重要配置

- `enable_context_injection`：是否每轮自动注入关系上下文，默认开启。
- `enable_dialogue_summary`：是否每隔几轮对话自动总结并写入关系/画像，默认关闭。
- `自动总结_触发轮数`：自动总结触发轮数，默认 6。
- `自动总结_模型Provider`：自动总结使用的 LLM Provider，留空使用当前会话模型。
- `自动总结_人格选择`：自动总结时参考的 AstrBot 人格，只用于理解说话风格和称呼，不会直接写成记忆。
- `自动总结_最大对话长度`：每次总结传给模型的最近对话最大长度，默认 4000 字。
- `记忆隔离_允许跨群召回`：工具查询时是否允许补充该用户其他群空间里的记忆。
- `私聊_同步到所属群空间`：私聊画像是否同步到该用户已出现过的群空间。
- `群成员目录_初始化获取`：新群空间首次触达时是否尝试获取群成员目录。
- `群成员目录_写入时校验`：写入画像和带 user_id 的关系前是否校验群成员目录。
- `记忆写入_关系必须包含群成员ID`：关系写入时是否要求至少带一个群成员 user_id。
- `enable_session_identity_injection`：是否注入当前群聊/私聊身份说明。
- `bot_relation_aliases`：机器人在关系记忆里的可能称呼。
- `relation_admin_user_ids`：允许使用 `/关系` 调试指令的用户 ID。
- `allow_public_debug_commands`：是否允许所有人使用 `/关系` 调试指令，默认关闭。
- `enable_person_profile`：是否生成当前发言人的简易画像，默认开启。
- `记忆管理_每轮注入关系数量`：每轮最多注入多少条相关关系，默认 5。
- `记忆管理_每人关系召回上限`：当前发言人每轮最多召回多少条直接关系，默认 8。
- `记忆管理_当前发言人画像条数`：当前发言人普通画像事实最多注入多少条，默认 4。
- `记忆管理_相关人物画像注入人数`：每轮最多注入多少个相关人物画像，默认 4。
- `记忆管理_画像查询返回人数`：画像查询最多返回多少个用户，默认 5。
- `记忆管理_每人记忆上限`：每个用户最多保留多少条普通画像事实，默认 20。
- `记忆管理_每轮记忆更新操作上限`：每轮总结最多执行多少条记忆更新，默认 6。
- `记忆管理_写入可信度阈值`：总结/抽取结果低于该可信度时不写入，默认 0.65。
- `群身份_初始化扫描`：是否在初始化时归纳群主/管理/群友身份。
- `enable_tool_read`：是否允许 LLM 主动查询群人物上下文，默认开启。
- `enable_tool_write`：是否允许 LLM 主动写入关系和画像，默认关闭。
- `enable_tool_update`：是否允许 LLM 主动修改/删除关系和基础画像，默认关闭。
- `enable_auto_extract`：是否自动从群聊消息抽取关系，默认关闭。

## 云云的使用建议

先开自动注入和工具读取，确认模型不会乱讲之后，再考虑打开写入。删除和修改能力更要慢点开，尤其是群友爱开玩笑的时候，模型太认真就会把破梗记进脑子里，真的挺烦。

基础画像适合长期稳定的信息，比如“以后叫我云子”“我喜欢打瓦”“我讨厌香菜”。普通画像事实适合更零散但仍然长期有用的内容。临时心情、当天安排、玩笑、攻击性评价和没确认的传闻，云云都建议别记。
